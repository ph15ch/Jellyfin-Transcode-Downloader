## Project Overview

A Jellyfin **C# server plugin** that adds a quality-selection download button to item
detail pages. The plugin embeds a client-side script (`web/plugin.js`), serves it from
an API controller, and injects a one-line `<script>` tag into the web client's
`index.html`. Jellyfin handles transcoding server-side; the client script is purely UX.

**Build:** `dotnet build`/`dotnet publish` against the Jellyfin SDK. Target framework is
**net9.0** (Jellyfin 10.11.x runs on .NET 9). The plugin ships as a single DLL.

## Architecture

### Client-script delivery (the important part)

The script is **never written into Jellyfin's web root** (that directory is read-only on
package installs and caused the original "button never appears" bug). Instead:

1. **`QuickDownloadController`** serves the embedded `web/plugin.js` at
   `GET /QuickDownload/ClientScript` with `Content-Type: application/javascript`. It is
   `[AllowAnonymous]` because the browser loads it via a plain `<script>` tag with no
   API token.
2. **`StartupTask` → `WebAssetInjector`** injects a single tag
   `<script src="{basePath}/QuickDownload/ClientScript?v={assemblyVersion}" defer></script>`
   into `index.html`:
   - **Primary path (no disk writes):** reflectively locate the **File Transformation**
     plugin (`Jellyfin.Plugin.FileTransformation.PluginInterface.RegisterTransformation(JObject)`)
     and register an in-memory `index.html` transformation. File Transformation rewrites
     the HTML as it is served — survives Jellyfin web updates, auto-removed on uninstall.
     This is the supported path and requires the **File Transformation companion plugin**.
   - **Fallback (writable installs only):** if File Transformation is absent, patch
     `index.html` on disk (best-effort). On `UnauthorizedAccessException` it logs a loud,
     actionable error recommending File Transformation (>= v2.2.1.0).

The reflection contract (confirmed against `IAmParadox27/jellyfin-plugin-file-transformation`
and `n00bcodr/Jellyfin-JavaScript-Injector`): build a Newtonsoft `JObject` payload with
`id`, `fileNamePattern` (`"index.html"`), `callbackAssembly`, `callbackClass`,
`callbackMethod`; FT deserializes the served HTML into the callback's parameter type and
takes the returned string as the new contents. Our callback is
`QuickDownloadTransformation.IndexHtml(PatchRequestPayload)` returning the patched HTML.

### Base-path handling

If Jellyfin runs under a sub-path, the injected `src` must respect it. `StartupTask`
reads the network configuration's `BaseUrl` via `IConfigurationManager.GetConfiguration("network")`
(reflected, to avoid a Jellyfin.Networking dependency) and prepends it to the script src.

### Cache-busting

The `?v=` query is the plugin assembly version, set at release time via
`dotnet publish -p:Version=$VERSION`. Read by reflection in `WebAssetInjector.BuildScriptTag()`.

### Client logic (`web/plugin.js` — only its delivery is owned by C#; do not change its logic)

- **Download URL — transcoded** (`selectedBitrate > 0`):
  `/Videos/{itemId}/stream.mp4?MaxStreamingBitrate={bitrate}&VideoCodec=h264&AudioCodec=aac&MaxAudioChannels=2&Static=false&api_key={token}`
- **Download URL — original** (`selectedBitrate === 0`): `/Items/{itemId}/Download?api_key={token}`
- **Filename** (from `/Users/{userId}/Items/{itemId}`):
  - Movie: `"{Name} ({ProductionYear}).mp4"`
  - Episode: `"{SeriesName} S{SeasonNumber padded}E{EpisodeNumber padded} - {Name}.mp4"`
- **Progress estimation:** `durationSeconds = RunTimeTicks / 10_000_000`;
  transcode `estimatedBytes = ((bitrate + 128_000) * durationSeconds) / 8`;
  original `MediaSources[0].Size`. ±10–15% (VBR) — show `~` prefix.
- **Download flow:** `fetch()` + `response.body.getReader()` → progress bar → `Blob` →
  `URL.createObjectURL()` → `<a download>` → `revokeObjectURL()`; `AbortController` for
  cancel. Original-file downloads use a direct `<a href>` to avoid large RAM use.
- **UI injection:** `.mainDetailButtons` / `.detailButtons` (fallback chain), with a retry
  loop. Prints `[QuickDownload] plugin loaded` to the console on success.

## Release pipeline

Tag-driven (`.github/workflows/release.yml`): an annotated tag `vX.Y.Z` is the single
source of truth. The tag message is the changelog. The pipeline builds with
`dotnet publish -p:Version=$VERSION`, zips the DLL, creates a GitHub release, and prepends
a manifest entry to `repo/manifest.json` on `main`. `targetAbi` and the SDK package
versions are kept in lockstep (currently **10.11.8** / ABI **10.11.8.0**). Do not hand-edit
manifest version entries.

## Known Pitfalls

| Issue | Mitigation |
|---|---|
| Web root is read-only on package installs | Serve JS from a controller; inject via File Transformation (no disk writes) |
| File Transformation absent | Best-effort on-disk fallback with a loud, actionable log; recommend installing FT >= v2.2.1.0 |
| File Transformation API drift across versions | Reflection contract pinned to the proven `RegisterTransformation(JObject)` shape |
| Newtonsoft vs System.Text.Json | FT's entry point takes a Newtonsoft `JObject`; reference Newtonsoft.Json 13.0.1 (runtime-provided, `PrivateAssets=All`) |
| ABI bump binary-compat | Build against 10.11.8 SDK; net9.0; test plugin load after bumping |
| Jellyfin under a base path | Inject base-path-aware `src` (read network `BaseUrl`) |
| Controller auth | Endpoint is `[AllowAnonymous]` — a `<script>` load carries no token; otherwise it 401s and the button silently never appears |
| Button injection selectors may change | `web/plugin.js` tries multiple selectors with a retry loop |
| VBR bitrate variance | Show `~` in UI; ±10–15% is acceptable |
| Blob for large files uses RAM | Direct `<a href>` for original downloads |
| CORS | Non-issue — same origin as the Jellyfin web client |
